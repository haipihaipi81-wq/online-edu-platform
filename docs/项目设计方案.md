# 在线教育平台 — 项目设计方案

> **技术栈**：Spring Boot + Vue 3 + Element Plus + MyBatis-Plus + MySQL + Redis + JWT  
> **分工**：两人协作，每人负责 2 个功能模块

---

## 一、项目背景与意义

随着互联网技术的发展，在线教育已成为重要的教学辅助手段。本系统旨在构建一个集**课程学习**、**在线考试**、**用户管理**、**互动社区**于一体的在线教育平台，解决传统教学中"教、学、练、评、互动"割裂的问题。

**核心价值**：
- 学生端：自主选课、碎片化学习、在线练习考试、社区交流
- 教师端：发布课程、管理题库、组卷考试、查看学情
- 管理员端：系统配置、用户管理、数据监控

### M1：课程管理（负责人 A）

| 功能点 | 说明 | 技术要点 |
|--------|------|----------|
| 课程 CRUD | 课程名称、简介、封面、价格、状态（草稿/已发布/下架） | 富文本编辑器、图片上传 |
| 分类管理 | 课程分类树（如：编程 → Java → Spring Boot） | 树形数据结构、级联选择器 |
| 章节管理 | 每个课程包含多章，章下包含多个课时 | 拖拽排序 |
| 课时管理 | 视频上传、文档上传、富文本内容 | 断点续传、OSS 存储 |
| 课程搜索 | 关键词搜索、分类筛选、排序（最新/最热） | Elasticsearch 或 MySQL 全文索引 |
| 课程推荐 | 首页热门课程、相关课程推荐 | 基于标签/分类的协同过滤（简单版） |
| 学习进度 | 记录用户每个课时的学习状态（未学/学习中/已完成） | Redis 缓存进度 |

### M2：在线考试（负责人 A）

| 功能点 | 说明 | 技术要点 |
|--------|------|----------|
| 题库管理 | 单选题、多选题、判断题、简答题 | 题目分类、难度等级、标签 |
| 组卷策略 | 手动选题组卷 + 自动随机组卷（按题型/难度/数量规则） | 组合算法 |
| 考试配置 | 考试时长、开始/结束时间、及格分数、允许重考次数 | 定时任务控制考试状态 |
| 在线答题 | 答题界面（题号导航、倒计时、自动保存） | 前端定时器、localStorage 防丢失 |
| 自动评分 | 客观题系统自动判分，主观题教师批阅 | 答案匹配算法 |
| 成绩统计 | 个人成绩报告、全班成绩分布（柱状图/饼图） | ECharts 可视化 |
| 错题本 | 自动收录错题，支持重做 | — |

### M3：用户与权限（负责人 B）

| 功能点 | 说明 | 技术要点 |
|--------|------|----------|
| 注册登录 | 手机号/邮箱注册，支持验证码 | JWT 双 Token（Access + Refresh） |
| 角色管理 | 学生、教师、管理员三级角色 | RBAC 权限模型（用户→角色→权限） |
| 个人信息 | 头像上传、昵称、简介、密码修改 | 文件上传、BCrypt 加密 |
| 我的课程 | 学生：已购/已收藏课程列表 | 分页查询 |
| 学习记录 | 学习时长、最近学习、学习日历 | — |
| 教师工作台 | 我的课程、学生数据、考试批阅 | 数据聚合查询 |
| 管理后台 | 用户管理、课程审核、系统配置 | 权限校验拦截器 |

### M4：互动社区（负责人 B）

| 功能点 | 说明 | 技术要点 |
|--------|------|----------|
| 课程讨论 | 每个课程下的讨论区（发帖/回帖） | 嵌套评论结构 |
| 问答板块 | 提问、回答、采纳最佳答案 | — |
| 笔记分享 | 学生记录学习笔记，可设为公开/私密 | Markdown 编辑器 |
| 点赞收藏 | 对课程、笔记、评论点赞和收藏 | Redis 去重 |
| 消息通知 | 回复通知、点赞通知、系统通知 | WebSocket 实时推送 |

### 课程相关（负责人 A）

```
courses                 课程表
├── id, title, description, cover_image, price
├── teacher_id (FK → users), category_id (FK)

├── created_at, updated_at

categories              分类表
├── id, name, parent_id, sort_order

chapters                章节表
├── id, course_id (FK), title, sort_order

lessons                 课时表
├── id, chapter_id (FK), title, type (VIDEO/DOC/TEXT)
├── content (JSON: video_url/doc_url/richtext), duration

├── id, user_id, course_id, progress (0.00-100.00)
```

## 四、API 接口设计（RESTful 风格）

## 七、答辩准备要点

### 7.1 答辩 PPT 建议结构（15-20 分钟）

| 环节 | 时间 | 内容 |
|------|------|------|
| 项目背景 | 2min | 选题意义、解决什么问题 |
| 需求分析 | 2min | 用例图、角色划分 |
| 系统设计 | 3min | 架构图、技术选型、数据库 ER 图 |
| 功能演示 | 5min | 核心流程录屏（课程学习 → 考试 → 查成绩 → 社区讨论） |

| 总结展望 | 2min | 项目成果、遇到的困难与解决、未来优化方向 |

### 7.2 答辩常见问题准备

1. **为什么选这个技术栈？**
   - Spring Boot：生态成熟、适合企业级开发、学习资料多
   - Vue 3：组件化开发、响应式数据、生态完善
   - MyBatis-Plus：简化 CRUD、分页插件、条件构造器

2. **JWT 如何保证安全性？**
   - Access Token 短期有效（30min），Refresh Token 长期（7天）
   - Token 存储在 HttpOnly Cookie 或前端内存（非 localStorage）
   - 退出登录时 Redis 黑名单机制

3. **自动组卷算法怎么实现？**
   - 用户设定规则：题型分布 + 难度分布 + 总分
   - 从题库中按条件随机抽取，满足约束条件
   - 贪心 + 回溯：先按题型分组，每组按难度随机选，不够则回退调整

4. **如何防止考试作弊？**
   - 切屏检测（Page Visibility API）
   - 禁止复制粘贴
   - 考试时间到自动提交
   - 打乱题目顺序和选项顺序

5. **WebSocket 消息通知怎么实现的？**
   - Spring Boot 集成 STOMP over WebSocket
   - 用户登录后建立连接，订阅个人通知频道
   - 后端产生通知事件 → 推送至对应用户频道
   - 前端实时更新未读消息徽标

6. **两个人怎么协作的？**
   - Git 分支管理：每人独立分支，PR 合并
   - 前后端接口先约定再开发（接口文档先行）
   - 数据库由一人统一维护，另一人提供需求

### 7.3 技术加分项（选做）

- [ ] 视频播放使用阿里云/腾讯云视频点播（VOD），支持防盗链
- [ ] 接口文档使用 Swagger/Knife4j 自动生成
- [ ] 使用 Docker Compose 一键部署（MySQL + Redis + Nginx + 应用）
- [ ] 前端使用 Nginx 部署，配置 Gzip 压缩
- [ ] 使用 Redis 缓存热点课程数据，减少数据库压力
- [ ] 操作日志记录（AOP 切面 + 自定义注解）
